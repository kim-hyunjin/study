# Blog Creator Skill

이 스킬은 개발자의 프로젝트 결과물을 고품질의 기술 블로그 콘텐츠로 변환하는 과정을 자동화합니다.

## 핵심 워크플로우

### 1. 글감 추출 (Extraction)
프로젝트 전반을 스캔하여 다음 기준에 부합하는 글감을 찾아 `todo.md`에 추가합니다.
- `note/` 폴더 내의 상세 학습 기록
- `README.md`에서 더 깊은 기술적 설명이 필요한 부분
- 소스 코드 내의 핵심 로직이나 복잡한 아키텍처 (주석 분석)
- 최근 수정되었거나 구현이 완료된 기능

**실행 방법:** `scripts/find_blog_potential.cjs`를 실행하거나 직접 워크스페이스를 탐색하여 `todo.md`의 `## 📝 New Blog Ideas` 섹션을 업데이트합니다.

### 2. 포스트 생성 (Authoring)
추출된 글감이나 사용자가 지정한 주제로 독자에게 실질적인 도움을 주는 기술 포스트를 작성합니다.
- **템플릿 사용:** `assets/post_template.md`의 구조를 참고하되 반드시 해당 구조를 따를 필요는 없음. **독자 중심**의 정보를 제공합니다.
- **콘텐츠 구성:**
  - **개요:** 이 기술이 독자에게 왜 유용한지, 어떤 문제를 해결하는지 명확히 제시합니다.
  - **핵심 개념:** 단순히 "무엇을 했다"가 아니라, 기술의 동작 원리와 핵심 메커니즘을 상세히 설명합니다.
  - **실전 구현:** 독자가 자신의 프로젝트에 바로 적용할 수 있는 구체적인 가이드와 코드 스니펫을 포함합니다.
  - **트러블슈팅 및 팁:** 실무에서 겪을 수 있는 문제점과 해결책, 성능 최적화 팁 등 부가가치가 높은 정보를 제공합니다.
- **메타데이터:** MkDocs-Material 테마에 최적화된 Frontmatter를 포함합니다.

### 3. MkDocs 등록 (Integration)
작성된 포스트는 MkDocs-Material의 블로그 플러그인과 awesome-pages 플러그인을 통해 **완전 자동으로 등록**됩니다.
- **저장 위치:** 포스트는 원본 글감 파일(소스 코드 또는 노트)이 위치한 **해당 프로젝트 디렉토리** 내에 저장합니다. (예: `docs/backend/msa/2024-04-23-msa-guide.md`)
- **자동화 원리:** 
    - **Blog 타임라인:** 파일 상단의 Frontmatter에 `date: YYYY-MM-DD`가 포함되어 있으면, `blog` 플러그인이 `docs` 전체에서 파일을 찾아 'Blog' 탭에 자동으로 나열합니다.
    - **카테고리 메뉴:** `awesome-pages` 플러그인이 폴더 구조를 스캔하여 해당 카테고리(Backend, AI 등) 탭의 하위 메뉴에 포스트를 자동으로 추가합니다.
- **분류 및 관리:** `mkdocs.yml`을 수정할 필요가 없으며, 포스트 상단의 `categories`와 `tags`를 정확히 입력하는 것만으로 충분합니다.

## 권장 사항
- **Mermaid 활용:** 아키텍처 설명 시 Mermaid 다이어그램을 생성하여 시각적 이해를 돕습니다.
- **코드 스니펫:** 핵심 로직은 반드시 코드 스니펫과 함께 상세 설명을 덧붙입니다.
- **SEO:** 제목과 `description` 필드를 검색 최적화에 유리하도록 작성합니다.
