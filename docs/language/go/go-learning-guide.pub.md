# Go 언어 기초 및 웹 프로그래밍 실습 가이드

## 🚀 개요
Go는 Google에서 개발한 오픈 소스 프로그래밍 언어로, 단순하고 효율적이며 신뢰할 수 있는 소프트웨어를 구축하기 위해 설계되었습니다. 본 문서는 Go의 핵심 문법부터 강력한 동시성 제어 모델, 그리고 실무적인 웹 프로그래밍 및 테스팅 기법까지 정리한 종합 가이드입니다.

---

## 💡 1. Go 언어의 핵심 철학 및 기초 문법

### 단순함과 효율성
- **객체지향의 재해석:** 클래스와 상속 대신 **구조체(Struct)**와 **인터페이스(Interface)**를 통한 구성을 지향합니다.
- **포인터:** 메모리 주소에 직접 접근할 수 있지만, C와 달리 포인터 연산은 제한하여 안전성을 높였습니다.

### 제어 흐름 및 데이터 구조
- **슬라이스 (Slice):** 배열의 상위 호환으로, 동적으로 크기가 조절되는 유연한 배열입니다.
- **맵 (Map):** 키-값 쌍을 저장하는 내장 해시 테이블입니다.
- **에러 핸들링:** 별도의 예외 처리(try-catch) 대신 함수의 반환 값으로 에러를 명시적으로 처리하는 관례를 따릅니다.

---

## 🐹 2. 강력한 동시성 모델 (Concurrency)

Go의 가장 큰 특징은 **고루틴(Goroutine)**과 **채널(Channel)**을 이용한 경량 스레드 모델입니다.

### 고루틴 (Goroutine)
- OS 스레드보다 훨씬 가벼운 사용자 수준 스레드로, `go` 키워드 하나로 수만 개를 동시에 실행할 수 있습니다.

### 채널 (Channel)
- 고루틴 간에 데이터를 주고받는 통로입니다. "공유 메모리로 통신하지 말고, 통신으로 메모리를 공유하라"는 Go의 철학을 대변합니다.
- **Select:** 여러 채널 중 데이터가 준비된 채널을 선택하여 처리할 수 있습니다.

---

## 🌐 3. 웹 프로그래밍 실습 (Web Programming)

Go의 표준 라이브러리(`net/http`)만으로도 강력한 웹 서버를 구축할 수 있습니다.

### 주요 실습 내용
- **RESTful API:** JSON 기반의 CRUD API 설계 및 구현.
- **Middleware:** 데코레이터 패턴을 활용한 로깅, 인증 처리.
- **OAuth2:** Google 등 외부 인증 연동 실습.
- **EventSource (SSE):** 실시간 데이터 전송 기술 구현.
- **File Server:** 정적 파일 서비스 및 템플릿 엔진 활용.

---

## 🛠️ 4. 테스팅 및 도구 (Testing & Tooling)

- **내장 테스팅:** `testing` 패키지를 통해 단위 테스트와 벤치마크 테스트를 기본으로 지원합니다.
- **Go Modules:** 의존성 관리의 표준으로, `go.mod`와 `go.sum`을 통해 프로젝트 패키지를 관리합니다.

---

## 📝 배운 점 및 결론
- **단순함의 힘:** 복잡한 기능을 언어 차원에서 최소화함으로써 코드의 가독성과 유지보수성이 비약적으로 높아짐을 체감했습니다.
- **동시성 제어:** 고루틴과 채널을 통해 복잡한 비동기 로직을 동기식 코드처럼 직관적으로 작성할 수 있는 점이 매우 강력합니다.
- **실무 적합성:** 컴파일 속도가 빠르고 단일 바이너리로 배포가 가능하여 마이크로서비스(MSA) 환경에 최적화된 언어임을 이해했습니다.
