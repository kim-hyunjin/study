# HAProxy 부하 분산 기술 가이드 (강의 요약)

이 문서는 소프트웨어 부하 분산 장치인 **HAProxy**의 핵심 기능과 실습 내용을 정리한 요약본입니다.

---

## 1. HAProxy 개요
*   **정의**: 신뢰성 높고 고성능인 오픈 소스 소프트웨어 부하 분산 장치.
*   **동작 계층**: TCP(Layer 4) 및 HTTP(Layer 7) 계층에서 동작.
*   **특징**: 업계 표준으로 여겨지며, 설정이 쉽고 강력한 고가용성/라우팅 기능을 제공함.
*   **플랫폼**: 리눅스 전용 애플리케이션이나, Docker 등을 통해 윈도우/맥에서도 실행 가능.

---

## 2. 설정 파일(`config`) 구조
HAProxy의 동작은 설정 파일에 의해 결정되며, 크게 두 가지 섹션으로 나뉩니다.

| 섹션 | 하위 항목 | 설명 |
| :--- | :--- | :--- |
| **Global** | - | 전체 프로세스에 적용되는 매개변수 (예: `maxconn`) |
| **Proxies** | `defaults` | 프록시 설정에 적용할 기본 매개변수 (타임아웃 등) |
| | `frontend` | 클라이언트의 요청을 받는 소켓 및 처리 로직 정의 |
| | `backend` | 실제 요청을 처리할 서버 클러스터 목록 |
| | `listen` | 프론트엔드와 백엔드를 하나로 합친 간편 설정 |

---

## 3. 주요 부하 분산 기능

### 3.1 로드밸런싱 알고리즘
*   **라운드 로빈 (Round Robin)**: 순차적으로 요청을 배분.
*   **가중 라운드 로빈 (Weighted Round Robin)**: 서버의 처리 능력에 따라 가중치(`weight`)를 부여하여 배분 비율 조정.

### 3.2 고가용성 (Health Check)
서버 장애 발생 시 자동으로 감지하여 분산 대상에서 제외합니다.
*   **기본 체크**: `check` 키워드 사용.
*   **HTTP 체크**: `option httpchk GET /status`와 같이 특정 종단점을 호출.
*   **고급 체크**: `http-check expect string "Server is alive"`를 통해 응답 본문의 특정 문자열 확인.
*   **주기 설정**: `inter` 옵션으로 상태 확인 빈도 조절.

### 3.3 모니터링 (Stats Page)
*   **관리자 화면**: 83번 등의 특정 포트를 통해 대시보드 제공.
*   **정보**: 서버별 상태(녹색: 정상, 붉은색: 장애), 세션 유지 시간, 데이터 전송량 등 실시간 모니터링.

### 3.4 고급 라우팅 (ACL)
**접근 제어 목록(ACL)**을 사용하여 요청 내용(URL 경로 등)에 따라 동적으로 백엔드를 선택합니다.
*   예: `/even` 경로는 짝수 서버군으로, `/odd` 경로는 홀수 서버군으로 라우팅.

---

## 4. 네트워크 계층별 동작 차이 (L4 vs L7)

| 비교 항목 | Layer 7 (HTTP Mode) | Layer 4 (TCP Mode) |
| :--- | :--- | :--- |
| **인식 단위** | HTTP 요청 (Request) 단위 | TCP 연결 (Connection) 단위 |
| **분석 내용** | URL, 헤더, 쿠키 등 상세 분석 가능 | IP 및 포트 정보만 확인 |
| **동작 특징** | 브라우저 새로고침 시마다 다른 서버 배정 가능 | 동일 TCP 연결 내의 요청은 계속 같은 서버로 전송 |
| **성능/용도** | 세밀한 제어 필요 시 (웹 앱) | 단순하고 빠른 전달 필요 시 (DB 등) |

> **주의**: L4 모드(TCP)에서는 브라우저가 동일한 TCP 연결을 유지(Keep-Alive)하기 때문에, 새로고침을 해도 요청이 다른 서버로 분산되지 않을 수 있습니다. (새 연결을 위해 브라우저 재시작 필요)
