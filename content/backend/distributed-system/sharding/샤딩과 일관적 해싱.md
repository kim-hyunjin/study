## 1. 샤딩(Sharding)의 개념
샤딩은 거대한 데이터셋을 **'샤드(Shard)'**라고 부르는 작고 관리하기 쉬운 단위로 나누어 여러 머신에 분산 저장하는 기술입니다.

* **수평 확장(Horizontal Scaling):** 데이터 양이 증가하면 인스턴스를 추가하여 대응할 수 있습니다.
* **성능 향상:** 데이터를 메모리에 적재하기 쉬워져 대기 시간이 단축되고, 병렬 처리를 통해 처리량이 늘어납니다.
* **가용성 증대:** 특정 샤드에 문제가 생겨도 다른 샤드는 정상 작동하므로 전체 시스템의 중단을 막을 수 있습니다.



## 2. 데이터베이스 유형별 샤딩 방식
* **SQL (관계형):** * **수직 샤딩:** 열(Column)을 기준으로 분할합니다.
    * **수평 샤딩:** 레코드(Row)를 독립된 단위로 분할하여 서로 다른 인스턴스에 저장합니다. (일반적인 확장 방법)
* **NoSQL (비관계형):** 레코드가 이미 독립적이므로 그룹화하여 샤드에 배치하기가 훨씬 간단하고 확장에 유리합니다.

## 3. 주요 샤딩 방법 (알고리즘)
데이터를 어떤 샤드에 넣을지 결정하는 기준입니다.

| 방법 | 설명 | 장점 | 단점 |
| :--- | :--- | :--- | :--- |
| **해시 기반 (Hash-based)** | 샤딩 키에 해시 함수를 적용하여 나온 값으로 샤드를 결정 (예: `ID % N`) | 데이터를 여러 샤드에 **고르게 분산**시킴 | 키값이 가까워도 다른 샤드에 저장되므로 **범위 쿼리에 불리** |
| **범위 기반 (Range-based)** | 키값을 특정 범위(알파벳순, 날짜, 국가 등)로 나누어 저장 | **범위 기반 쿼리 효율**이 매우 좋음 | 특정 범위에 데이터가 몰릴 경우 **데이터 불균형** 발생 가능 |



## 4. 샤딩의 문제점과 단점
샤딩은 성능을 높여주지만, 시스템의 복잡도를 크게 증가시킵니다.

* **복잡한 동시성 제어:** 여러 샤드에 걸친 데이터를 다룰 때 데이터 일관성을 보장하기 어렵고 비용이 많이 듭니다.
* **분산 트랜잭션의 어려움:** 단일 위치에서 처리하던 SQL 트랜잭션과 달리, 여러 머신을 오가는 '분산 잠금'이나 복잡한 쿼리 처리가 필요하여 성능 저하가 발생할 수 있습니다.
* **NoSQL의 절충안:** 확장성이 뛰어난 NoSQL(DynamoDB, Cassandra 등)은 샤딩이 쉽지만, SQL만큼의 엄격한 ACID(원자성, 일관성 등)를 보장하지 않는 경우가 많아 설계 시 주의가 필요합니다.

---
**핵심 요약:**
샤딩은 대규모 데이터를 처리하기 위한 필수 기술이지만, **데이터 분산 방식(해시 vs 범위)**에 따라 성능 특성이 다르며, **데이터 일관성과 복잡성 사이의 트레이드오프(Trade-off)**를 신중히 고려하여 설계해야 합니다.

---

이번 강의는 해시 기반 샤딩의 한계를 극복하기 위한 핵심 알고리즘인 **'일관된 해싱(Consistent Hashing)'**에 대해 다루고 있습니다. 주요 내용을 체계적으로 정리해 드립니다.

---

## 1. 일반 해시 기반 샤딩의 문제점
단순히 `Hash(key) % N` 방식을 사용할 경우 두 가지 치명적인 결함이 발생합니다.

* **재배치(Rehash) 비용 발생:** 서버(N)를 추가하거나 제거하면 모든 공식이 바뀌어, 기존 데이터를 거의 대부분 새로운 서버로 옮겨야 합니다. 이는 시스템에 엄청난 부하를 줍니다.
* **하드웨어 불균형 대응 불가:** 모든 서버에 데이터를 균등하게만 나누려 하므로, 성능이 좋은 서버(큰 메모리/CPU)를 더 많이 활용하고 노후 서버를 적게 활용하는 식의 유연한 대처가 어렵습니다.

## 2. 일관된 해싱 (Consistent Hashing)
일관된 해싱은 **키(Data)**와 **노드(Server)**를 동일한 가상의 **해시 링(Hash Ring)** 위에 배치하는 방식입니다.

* **작동 원리:** 해시 공간을 고리 모양으로 만들고 노드들을 배치합니다. 특정 키는 링을 시계 방향으로 따라가다 만나는 첫 번째 노드에 저장됩니다.
* **노드 추가/삭제의 유연성:** 노드가 추가되거나 삭제될 때, **인접한 노드 간의 데이터만 이동**하면 됩니다. 링 전체의 데이터를 재배치할 필요가 없어 이동량이 최소화됩니다.



## 3. 고급 활용 기술

### ① 가상 노드 (Virtual Nodes)
물리적인 서버 한 대를 링 위의 여러 지점(가상 노드)에 배치하는 기술입니다.
* **장점:** 서버마다 가상 노드 개수를 다르게 설정하여 **하드웨어 성능에 따른 부하 조절**이 가능합니다. (성능 좋은 서버 = 가상 노드 많이 할당)

### ② 다중 해시 함수 (Multiple Hash Functions)
노드 수가 적을 때 특정 노드에 데이터가 몰리는 '불균형 배치' 문제를 해결하기 위해 사용합니다.
* **작동 원리:** 각 노드에 대해 여러 개의 해시 함수를 적용하여 링 위의 여러 위치에 분산 배치합니다.
* **효과:** 통계적으로 데이터 분포를 훨씬 더 고르게 만들어 특정 서버의 **성능 병목 현상을 방지**합니다.



---

**핵심 요약:**
일관된 해싱은 서버의 동적인 변화(추가/제거)에도 **최소한의 데이터 이동**만으로 시스템을 유지하게 해주며, **가상 노드**와 **다중 해시** 기법을 통해 다양한 성능의 서버를 효율적으로 혼합하여 사용할 수 있게 해주는 분산 시스템의 필수 알고리즘입니다.
